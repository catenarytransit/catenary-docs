# Flujo de trabajo

Permítenos presentarte el flujo de trabajo de Catenary. ¡Asegúrate de leer la página de GitHub si eres nuevo en GitHub!

Como se indicó anteriormente, Catenary utiliza GitHub para administrar nuestro código base; también es lo que nos hace de código abierto: ¡cualquiera puede ver y copiar nuestro código! Sin embargo, para agregar a nuestra rama oficial, requerimos que los colaboradores realicen solicitudes de extracción a través de GitHub como un medio para promover la comunicación y las mejoras para nuestro código base operativo. (Nota: si te abres camino como colaborador habitual, ¡podemos otorgarte permisos de push-to-main!)

Si no te has unido a nuestro Discord, se recomienda encarecidamente que lo hagas, ¡porque es una línea directa con los otros desarrolladores! Nuestros Guías son muy activos en nuestro servidor y responden rápidamente. Los Guías son mantenedores que han estado trabajando en estrecha colaboración con cierto(s) proyecto(s) de Catenary y están disponibles para ayudar a las personas. Si haces una Solicitud de Extracción, publicas en discusiones en GitHub o haces ping a **@Guides** en Discord, ¡ellos serán los que respondan! A partir de octubre de 2024, estos Guías son:

Catenary no requiere ni tiene reuniones periódicas como clubes escolares u organizaciones profesionales; tratamos de mantener todo asíncrono ya que tenemos miembros de una gran variedad de orígenes y zonas horarias. ¡*Sí* tenemos eventos tipo ayuntamiento donde repasamos temas relevantes para nuestro desarrollo y actividades, y los equipos más pequeños pueden decidir programar discusiones también! Además, si ves a alguien en nuestros canales de voz de Discord, siéntete libre de unirte y escuchar y/o hablar (con respeto, por supuesto): ¡estamos felices de explicar en qué estamos trabajando o simplemente charlar con nuestra comunidad!

También puedes intentar contactarnos en nuestras diversas cuentas de redes sociales. Si estás aquí, ¡es muy probable que nos hayas encontrado a través de una de ellas!
